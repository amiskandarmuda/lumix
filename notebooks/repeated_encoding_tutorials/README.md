# Repeated Encoding Tutorials

This folder contains tutorial notebooks for the Lumix repeated data re-encoding experiments.

## Notebooks

1. `01_repeated_encoding_basics.ipynb`  
   Introduces phase masks, repeated encoding, a small numerical example, and the Flax module shape.

2. `02_mnist_pca_phase_dataset.ipynb`  
   Builds and visualizes the MNIST 16-component PCA phase-only input representation.

3. `03_mnist_repeated_vs_williamson.ipynb`  
   Compares unitary repeated, subunitary repeated, and Williamson models using cached depth-curve results.

4. `04_objective_and_readout_ablation.ipynb`  
   Compares normalized-intensity, fixed-temperature softmax, trainable-temperature softmax, and ridge readout.

5. `05_imagenette_repeated_optical_network.ipynb`  
   Explains the Imagenette patch/time-multiplexed optical pipeline and Flax architecture.

6. `06_imagenette_parameter_sharing_generalization.ipynb`  
   Shows Imagenette parameter sharing as a regularization and hardware-simplification story.

7. `07_mnist_local_routing_93_accuracy.ipynb`  
   Trains the local-routing repeated subunitary MNIST model with `routing_limit=4`, visualizes routing leakage and insertion loss, and reproduces the `>93%` validation accuracy result.

## Environment

Install the tutorial extras before running the notebooks:

```bash
uv pip install -e ".[tutorial]"
```

The notebooks use notebook-owned caches under `notebooks/repeated_encoding_tutorials/cache/`. If a cache is missing, the corresponding notebook regenerates it from deterministic notebook cells before plotting.

## Determinism

Each notebook defines `RNG_SEED = 7` in its setup cell and uses that seed for NumPy and JAX examples. Cached benchmark results are generated by the notebooks themselves.

Current notebook-owned caches:

- `mnist_pca16_phase_no_clip_rng7.npz`
- `mnist_three_way_depth_curve_rng7.json`
- `mnist_objective_readout_rng7.json`
- `mnist_ridge_readout_rng7.json`
- `mnist_local_routing_limit4_rng7.json`
- `imagenette_sharing_rng7.json`

Each JSON cache records its notebook `source` and `rng_seed`. The PCA `.npz` cache records the same metadata.

To regenerate notebook outputs in place:

```bash
for nb in notebooks/repeated_encoding_tutorials/*.ipynb; do
  uv run jupyter nbconvert --to notebook --execute --inplace "$nb" --ExecutePreprocessor.timeout=600
done
```
